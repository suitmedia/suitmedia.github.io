Avoid deep nesting

Maksimal nesting adalah 3 level, lebih dari itu hindari. Nesting level paling akhir apabila adalah Pseudo Class atau Pseudo Element diperbolehkan.

// Bad
ul {
    li {
        a {
            span {
                ...
            }
        }
    }
}

// Good
ul {
    li {
        a {
            ...
        }
    }
}

ul {
    li {
        a {
            &:hover {
                ...
            }
        }
    }
}

Z-Index

Maksimal nilai z-index adalah 20, tidak boleh lebih.

header {
    position: relative;
    z-index: 2;
}

main {
    position: relative;
    z-index: 1;
}

.modal {
    position: absolute;
    z-index: 20;
}

Spacing and new lines

Agar lebih mudah dibaca dan maintainable, tambahkan space dan new line secukupnya.

// Bad
a{
    prop: value;
    &:hover,&:focus{
        prop: value;
    }
    &:active{
        prop: value;
    }
    span{
        color: #999;
    }
}
button{
    ...
}

// Good
a {
    prop: value;

    &:hover,
    &:focus {
        prop: value;
    }

    &:active {
        prop: value;
    }

    span {
        color: #999;
    }
}

button {
    ...
}

CSS Prefixing

CSS prefixing di-handle oleh autoprefixer, jadi tidak perlu menambahkan vendor-prefix secara manual atau menggunakan mixin.

Perlu diketahui autoprefixer hanya meng-parse CSS property official yang telah ditetapkan oleh W3.. Jadi property appearance apabila memang ingin digunakan, harus input vendor prefix secara manual.

Khusus gradient, ikuti deklarasi yang telah ditetapkan oleh W3.

.box {
    background-image: linear-gradient(to bottom, #fff, #eee);
    transform: scale(1);
    transition: all .3s ease;
}

No ID and !important in CSS

Jangan ada deklarasi ID ataupun !important di dalam CSS karena ID memiliki tingkat specificity yang tinggi.

Apabila memang terpaksa harus menggunakan, gunakan attribute selector. Ini adalah sebuah hack dan sebisa mungkin jangan sampai ada.

Penggunakan !important hanya diperbolehkan untuk override style 3rd party ataupun style plugin.

/* NO! */
#myElem {
    prop: value;
}

button {
    padding: .5em 1em !important;
}

/* YES! */
[id="myElem"] {
    prop: value;
}

button {
    padding: .5em 1em !important;
}

Avoid contextual styles

Intinya jangan buat style yang medeskripsikan untuk apa atau untuk siapa style itu ditujukan.

Styles harus dibuat se-abstract mungkin agar dapat dipakai berkali-kali dan dapat dipakai untuk segala konteks.

/* NO! */
.btn-download {
    prop: value;
}

/* YES! */
.btn-primary {
    prop: value;
}

/* NO! */
.faq-wrapper {
    width: 400px;
}

/* YES! */
.container {
    width: 800px;
}

.container--small {
    width: 400px;
}

Don't change base style

Jangan menambahkan/mengubah behavior style yang sudah dimiliki tiap element. Apabila ingin melakukan itu, gunakan class.

Hal ini untuk mencegah hal yang tidak diinginkan apabila user meng-input content dari editor.

Kalaupun memang ada style khusus untuk tiap-tiap element contoh untuk article, wrap dengan sebuah class lalu declare contextual style berdasarkan wrappernya.

// NO!
h1 {
    text-decoration: underline;
    text-transform: uppercase;
}

textarea {
    min-height: 90px;
    -webkit-appearance: none;
}

ul {
    width: 400px;
}

// YES!
.heading {
    text-decoration: underline;
    text-transform: uppercase;
}

.field {
    min-height: 90px;
    -webkit-appearance: none;
}

.list {
    width: 400px;
}

//Contextual Style
.article {

    h1 {
        text-transform: uppercase;
    }

    p {
        font-size: 13px;
        font-weight: 300;
    }

}

Use class to style, avoid using tag.

Sebisa mungkin hindari memberi style menggunakan tag element. Gunakan class agar style yang dibuat scalable dan lebih mudah di maintain.

Selain itu apabila ke depannya markup diubah, artinya kita harus mengubah css kita juga. Dengan menggunakan class, walau markup berubah, tidak akan mempengaruhi style.

/* NO! */
.highlight {
    float: left;
    width: 30%;
    margin: 0 3%;

    h2 {
        text-transform: capitalized;
    }

    ul {
        color: #444;
    }
}

/* GOOD! */
.highlight {
    float: left;
    width: 30%;
    margin: 0 3%;
}

.highlight-title {
    text-transform: capitalized;
}

.highlight-list {
    color: #444;
}

Don't use BEM SCSS shortcut

Memang memudahkan jadi tidak perlu menulis berulang-ulang, namun itu membuat file .scss susah untuk di index/search dan akan merepotkan developer penerus.

/* NO! */
.highlight {
    ...

    &__title {
        ...
    }

    &__list {
        ...
    }
}

/* YES! */
.highlight {
    ...
}

.highlight__title {
    ...
}

.highlight__list {
    ...
}

Don't CSS use @import

Alasan kenapa jangan menggunakan @import adalah:

  • File CSS yang dipanggil via @import tidak di cache di browser.
  • File CSS yang dipanggil via @import tidak dapat didownload oleh browser secara paralel.
  • Mengurangi performance web.
/* NO! */
@import url('hello.css');
  
<!-- YES! -->
<link rel="stylesheet" href="hello.css">
<!-- or use scss import -->

Use transform and opacity for high performance animation

Untuk membuat animasi contoh pada saat image di hover, image membesar, jangan gunakan width dan height.

Gunakan property transform agar dapat dirender menggunakan GPU, bukan CPU. Pada dasarnya, untuk mendapatkan high performance animation, gunakan:

  • Translate
  • Scale
  • Rotate
  • Opacity
/* NO! */
div {
    position: relative;
    top: 0;
    transition: all .3s ease;

    &:hover {
        top: 100%;
    }
}

/* YES! */
div {
    trasnform: translateY(0);
    transition: all .3s ease;

    &:hover {
        transform: translateY(100%);
    }
}
